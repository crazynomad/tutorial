# Claude Code + Skill vs OpenClaw

## Overview
两种"让 AI 动手"的路线对比——稳健可控 vs 激进接管

## Learning Objectives
The viewer will understand:
1. 两者的核心运行方式差异（终端文字 vs 屏幕操控）
2. 安全性与适用场景的权衡
3. 共同代表的趋势：AI 从对话到行动

---

## Left Column: Claude Code + Skill

**Headline**: "🤖 Claude Code + Skill"
**Subhead**: "稳健可控路线"

**Rows**:
- 运行方式: 终端命令行，文字交互
- 操作模式: 每一步经过用户确认，Skill 是预定义工作流
- 文件访问: 直接读取本地文件系统，文件就是上下文
- 扩展性: 安装 Skill 扩展能力，像给操作系统打补丁
- 安全性: 不直接控制鼠标键盘，操作透明可审计
- 适合场景: 重复性工作流自动化（磁盘清理、文件整理、文档分析）

**Tagline**: "你说人话，它办实事"

---

## Right Column: OpenClaw

**Headline**: "🦞 OpenClaw"
**Subhead**: "激进接管路线"

**Rows**:
- 运行方式: 直接操控屏幕和鼠标，视觉交互
- 操作模式: 自主决策，像坐在你电脑前的助手
- 多平台: WhatsApp / Telegram / Discord / Slack / Signal / iMessage
- 扩展性: 社区 Skills 插件无限扩展
- 安全性: 直接接管屏幕和鼠标（需信任）
- 适合场景: 需要 GUI 操作的复杂任务，跨应用协作

**Tagline**: "The AI that actually does things."
**Badge**: GitHub 100,000+ Stars

---

## Bottom Bar: 共同点

- 都是本地运行的 AI 助手
- 都支持 Skills 扩展
- 都能操作你的电脑完成实际任务
- 代表 AI 从"对话"到"行动"的进化方向
