# Claude Code + Skill vs OpenClaw 对比

## Claude Code + Skill（稳健可控路线）
- 运行方式：终端命令行，文字交互
- 操作模式：每一步都经过用户确认，Skill 是预定义的工作流
- 文件访问：直接读取本地文件系统，文件就是上下文
- 扩展性：通过安装 Skill 扩展能力，像给操作系统打补丁
- 安全性：不直接控制鼠标键盘，操作透明可审计
- 适合场景：重复性工作流自动化（磁盘清理、文件整理、文档分析）
- 核心理念：你说人话，它办实事（稳健可控）

## OpenClaw（激进接管路线）
- 运行方式：直接操控屏幕和鼠标，视觉交互
- 操作模式：自主决策，像一个坐在你电脑前的助手
- 多平台：支持 WhatsApp/Telegram/Discord/Slack/Signal/iMessage
- 记忆：跨对话持久记忆，记住用户偏好
- 扩展性：社区 Skills 插件无限扩展
- 适合场景：需要 GUI 操作的复杂任务，跨应用协作
- 核心理念：The AI that actually does things（激进但强大）
- GitHub 100,000+ Stars

## 共同点
- 都是本地运行的 AI 助手
- 都支持 Skills 扩展
- 都能操作你的电脑完成实际任务
- 代表 AI 从"对话"到"行动"的进化方向
