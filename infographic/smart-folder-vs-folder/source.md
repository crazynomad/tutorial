# macOS 智能文件夹 vs 传统文件夹

## 核心类比：书架 vs 书单

### 传统文件夹 = 书架
- 一本书只能放在一个书架上（文件物理存放在某个目录里）
- 你把书从 A 架搬到 B 架，A 架就空了
- 删掉书架上的书 = 书真的没了
- 删掉书架 = 书也跟着没了

### 智能文件夹 = 书单（读书清单）
- 你写了一条规则，比如"列出所有超过 1GB 的视频文件"
- 符合条件的文件自动出现在里面，文件还在原来的位置
- 新下载的文件如果符合条件，自动上榜；不符合的自动消失
- 删掉这张清单，文件一个都不会少
- 同一个文件可以出现在多个智能文件夹中

## 技术对比

| 维度 | 传统文件夹 | 智能文件夹 |
|------|-----------|-----------|
| 本质 | 目录（容器） | 保存的搜索条件（.savedSearch 文件） |
| 文件位置 | 文件在文件夹里 | 文件留在原处，只是匹配显示 |
| 移入/移出 | 手动拖拽 | 自动，改条件即变 |
| 删除文件夹 | 里面的文件跟着删 | 文件不受影响 |
| 一个文件能否出现在多处 | 不能（除非别名/符号链接） | 能，只要满足多个条件 |

## 一句话总结
传统文件夹是"把东西放进去"，智能文件夹是"把条件写出来"。
