## 🎯 制作流程

基于第二大脑的 **C.O.D.E 框架**进行知识管理和内容创作：

### 1. 收集 (Capture) - 多源信息导入

**NotebookLM 支持的导入格式**：
- 📄 文档：PDF, Markdown, Word, Google Docs
- 🌐 网页：直接粘贴 URL，自动抓取内容
- 🎥 视频：YouTube 链接，自动提取视频内容
- 🎵 音频：音频文件上传
- 🖼️ 图片：直接导入图片文件
- 🔍 搜索：内置搜索功能，快速找到相关资料

**操作步骤**：
1. 创建新笔记本（New Notebook）
2. 从官方文档开始，确保信息准确性
3. 添加技术博主视频，获取实践经验
4. 补充搜索结果，扩展知识面
5. 所有材料自动归档到 `downloads/`

### 2. 整理 (Organize) - 结构化信息梳理

**核心功能**：
- 📊 **信息图生成**：系统自动分析内容关联性，生成可视化结构
- 🎯 **自定义结构**：使用 5W1H 等框架定制输出格式
- 🔗 **智能关联**：跨文档理解，串联不同来源的信息

**操作步骤**：
1. 点击"生成信息图"按钮
2. 使用自定义提示词（参考 `prompt/5w1h.md`）
3. 调整结构和布局
4. 导出到 `artifact/` 目录

### 3. 消化 (Distill) - 深度理解与内化

**学习工具**：
- 📚 **学习指南**：AI 私教模式，提供结构化学习路径
- 🎙️ **播客音频**：双人对谈形式，轻松理解复杂概念
- 💬 **智能问答**：针对特定来源深度追问
- 📝 **笔记保存**：重要回答保存到笔记本

**操作步骤**：
1. 切换到"学习指南"模式
2. 跟随 AI 生成的学习路径
3. 生成播客音频（支持后台播放）
4. 针对疑问进行深度提问
5. 保存关键笔记和见解

### 4. 表达 (Express) - 多形式内容输出

**自动生成内容**：
- 📊 **演示文稿**：专业技术分享格式
- 📱 **小红书图文**：社交媒体传播格式（3:4 竖版）
- 📹 **视频预览**：带语音讲解的演示
- 📄 **学习笔记**：结构化知识整理

**操作步骤**：
1. 点击"生成演示文稿"
2. 使用自定义提示词调整风格（参考 `prompt/rednote.md`）
3. 导出多种格式内容到 `artifact/`
4. 使用 ScreenFlow 录制操作过程 → `recording/`
5. 使用剪映进行后期制作 → `capcut/`
6. 导出最终视频 → `output/`
7. 提取字幕 → `transript/`

### 5. 发布优化与多平台分发

**视频格式适配**：
- 📺 **16:9 横屏**：适合 Bilibili、YouTube 等长视频平台
- 📱 **9:16 竖屏**：适合抖音、小红书、微信视频号等短视频平台

**内容形式矩阵**：
- 🎥 **视频内容**：长视频（教学详解）+ 短视频（快速传播）
- 📝 **图文内容**：小红书图文（3:4 竖版，使用 NotebookLM 提示词生成）
- 📄 **文档内容**：演示文稿、学习笔记

**发布平台策略**：
- **长视频平台**：Bilibili（技术社区）、YouTube（国际观众）
- **短视频平台**：抖音（大众传播）、小红书（图文+视频）
- **私域平台**：微信公众号（深度内容）

**发布优化要点**：
- 根据平台特性优化标题、封面和内容结构
- 小红书采用"反共识"标题吸引注意力

## 💡 NotebookLM 核心价值

### 解决大模型的两大硬伤

1. **AI 幻觉问题**
   - ❌ 传统 AI：不知道答案时会一本正经地编造
   - ✅ NotebookLM：严格基于用户提供的材料，每句回答带引用出处

2. **上下文限制**
   - ❌ 传统 AI：处理长文档时只能看前面几十页，后面容易遗漏或混淆
   - ✅ NotebookLM：无限制消化 PDF、视频、网页等内容

### 第二大脑的未来形态

根据"第二大脑"概念提出者 Thiago Forte 的设想：
- **传统工具的困境**：Notion、Obsidian 等工具门槛高，容易变成"积灰的收藏夹"
- **NotebookLM 的革新**：把最费劲的"整理"和"关联"工作交给 AI
  - 你只管提问 → AI 负责思考
  - 你只管投喂 → AI 负责消化
  - 你只管表达 → AI 负责生成

### 与传统 AI 的核心区别

| 维度 | 传统大模型 | NotebookLM |
|------|-----------|------------|
| 信息来源 | 训练数据（可能过时） | 用户提供的精准素材 |
| 回答准确性 | 可能产生幻觉 | 基于材料，有据可查 |
| 内容长度 | 受上下文窗口限制 | 无限制处理 |
| 多源整合 | 需要手动输入 | 自动跨格式理解和关联 |
| 输出形式 | 主要是文本对话 | 信息图、播客、演示文稿等多形式 |

## 📋 内容创作方法论

### 5W1H 分析框架
结构化分析 AI 技术和产品的方法：
- **What**: 是什么
- **Why**: 为什么重要
- **Who**: 谁应该用
- **When**: 什么时候用
- **Where**: 在哪些场景中用
- **How**: 怎么用才是最佳实践

详见：[prompt/5w1h.md](prompt/5w1h.md)

### 小红书内容创作
针对知识型内容的图文创作指南：
- 先给结论，再解释原因
- 讲人话，不讲术语
- 场景化表达
- 3:4 竖版图片（1080×1440px）

详见：[prompt/rednote.md](prompt/rednote.md)

## 🔍 NotebookLM 深度见解

### Skills vs Agents 的关系

根据视频中的深度分析：
- **Agents**：通用化的执行主体（智商 300 的天才）
- **Skills**：插件式的专业技能包（天才身边的专业顾问）
- **关系**：一个通用 Agent 通过加载不同 Skills，瞬间变成各领域专家

### Skills 相比 MCP 的优势

视频中指出的关键痛点：
1. **MCP 的问题**：
   - 加载几个复杂 MCP 就消耗几百 K 上下文
   - 疯狂消耗 Token，经常达到 quota 上限导致 Claude Code 罢工

2. **Skills 的解决方案**：
   - **按需加载**：只在需要时才调用，不占用常驻上下文
   - **分层加载策略**：确保资源高效利用
   - **标准化设计**：像乐高一样可组合、可移植

### NotebookLM 实际应用场景

1. **新技术学习**：快速构建专家知识库（如 Claude Skills）
2. **研究分析**：整合多源信息，生成结构化报告
3. **内容创作**：自动生成演示文稿、播客、图文内容
4. **知识管理**：打造个人第二大脑，避免信息孤岛

## 💎 使用技巧与最佳实践

### 信息收集阶段
1. **从源头找起**：优先使用官方文档，确保信息准确性
2. **精不贵多**：重质量而非数量，避免信息过载
3. **多样化来源**：结合官方文档、技术博客、视频教程

### 信息整理阶段
1. **自定义结构**：使用 5W1H、SWOT 等框架定制输出
2. **分层理解**：先看概览信息图，再深挖细节
3. **针对性提问**：限定特定来源范围，避免信息混淆

### 内容消化阶段
1. **学习指南模式**：初期使用 AI 私教引导学习路径
2. **播客音频**：通勤时听播客，提升学习效率
3. **保存关键笔记**：及时保存有价值的回答和见解

### 内容输出阶段
1. **以教促学**：通过分享来验证理解程度
2. **多形式输出**：根据受众调整内容形式（技术分享 vs 社交传播）
3. **迭代优化**：根据反馈不断改进提示词和输出质量
