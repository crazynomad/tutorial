# 安全区契约 (Safe Zone Spec)

> **第一原则**：前景文字只能排在图的天然安全区里。装不下 → `slide.addNotes()`，**不许缩字号**。

## 三种标准 layout 的安全区

### HF (Hero-Fill, 16:9 全幅背景)
用于封面、章节页、封底、大金句页。

- **背景图**：16:9 全幅铺满（720pt × 405pt）
- **部署加 62% 暗罩**：`rgba(10,10,10,0.62)` 或 `0.55`，保证文字对比度
- **天然安全区**：图像的 top 15% / bottom 15% 要求是 void black，能放大标题、label、source
- **文字规则**：hero 标题 ≤ 60pt，副标题 ≤ 16pt，label ≤ 9pt；整体文字密度不超过画面 30%

### R34 (Right 3:4, 右侧竖图 + 左侧文字)
用于内容页，适合有一个核心视觉隐喻的论点页。

- **图像**：右侧 304pt × 405pt（3:4 比例）
- **文字安全区**：左侧 padding 48pt，右边界 **不越过 380pt**（给竖图留 16pt 气口）
- **图像柔化**：图像左缘 40pt 渐变到底色，避免硬切
- **文字容量**：eyebrow 9pt + 标题 32pt（≤2行）+ lede 14pt（≤2行）+ 3~4 条 bullet (12pt)

### L34 (Left 3:4, 左侧竖图 + 右侧文字)
R34 的镜像版本。图在左，文字在右。

- `.image-pane` 改 `left:0`，渐变方向反转
- 正文 padding 改为 `42pt 48pt 36pt 320pt`（左 padding 给图让位）

## 如何判断"装不下"

`html2pptx.js` 会**自动检测 overflow** —— body 的 `scrollHeight/scrollWidth` 超过声明尺寸 > 1px 就抛错。

但 overflow 报警只是底线。真正的判断：**在 Chrome 浏览器里打开 HTML，按 `720pt × 405pt` 预览**（或 1280×720 / 960×540 按比例），目测：
- 文字是否压到图像主体？→ 删内容
- 文字是否贴边 < 24pt？→ 删内容
- 视觉重心是否集中在图像安全区外？→ 重排

**关键心态**：PPT 不是文档。一页放不下的重点是 deck 结构问题，不是排版问题。

## 反例（绝不要做）

❌ **把字号从 12pt 压到 9pt 硬塞** — 观众看不清，视觉破碎
❌ **把 HF 暗罩调到 80%+ 盖住整张图** — 图就白生成了，等于没图
❌ **在 R34 的图像上压文字** — 除非图特意留了中心 void 区
❌ **同一页放 >5 个 bullet** — 超过就该拆页或进 notes

## 溢出走 notes 的正确姿势

```js
// notes-map.js
module.exports = {
  12: `【口播 · 第 12 页】
— 这条信息在画面上太挤，放到口播
— 补充数据 / 背景故事 / 引用链接`,
};
```

观众看画面、演讲者看 notes、读者看 PDF 时画面 + notes 都有。三赢。
