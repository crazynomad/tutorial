## 第 0 章 · 为什么不用一键工具

做 PPT 最大的错误，是先打开 PowerPoint。

我们见过太多人用 Canva / Gamma / 各种"AI 一键生成"工具，30 秒出一份「看起来很专业」的 PPT。等到这份 PPT 拿到高管会议、客户提案、研究分享的场子，第一眼就穿帮。

**为什么穿帮？** 因为一键工具替你做了所有决策。但专业方案的「专业」两个字，恰恰就在这些决策里：

- 这是哪种类型的方案？（立论 / 研究 / 教学 / 叙事 —— 用错类型每页都不成立）
- 哪条证据是可信的？哪条只是道听途说？
- 哪个骨架对题？哪条路径最锐？
- 什么场合用全画幅？什么场合用分栏？

**这些判断没法外包**。AI 是这齣戏里的演员，你才是导演。

## 第 2 章 · 三个 Skills 详解

### 2.1 · `/ppt-classify` — 定调

**它解决什么问题**：让你不在错误的框架里浪费时间。

四种类型的论述结构**完全不同**：

| 类型 | 开场 | 结构 | 结论 |
|---|---|---|---|
| 立论型 | 我主张什么 | 论据堆叠 | 可信结论 |
| 研究型 | 反共识悖论 | 逐条展开 + 分析 | 可证伪双向信号 |
| 教学型 | 学完后能做什么 | 步骤递进 | 可复现 demo |
| 叙事型 | 一个具体场景 | 时间线 / 角色弧 | 留白让观众自填 |

用错类型 = 用爱情片的剧本拍悬疑片。每页都努力，每页都不成立。

**输入 4 项**：题目 / 素材链接 / 目标受众 / 时长预期。

**输出**：类型判定 + 按题目定制的章节骨架（**不是模板**，结合你的题目在选定类型下给出的起手结构）。

**关键心智**：分类给的是起点，不是终点。骨架在后面 review 阶段还会改。

---

### 2.2 · `/ppt-research-setup` — 填证据

**它解决什么问题**：让你不在没有证据的情况下写论证稿。

**前置动作**：先在 NotebookLM 里**针对每一章**拉一手证据，每章 3-5 条，汇总成 evidence 文档。

**这一档的核心动作**：把 evidence 喂给 Skill，让它按章节输出：

- **Section 1（反共识悖论）**：候选 + 具体措辞
- **Section 2（被研究对象的论证）**：先不反驳，建立对方逻辑链 —— 这是研究型的专业核心
- **Section 3（调查路径）**：直观层 / 反推层 / 盲区层

**关键心智**：这一档产出的是初稿。结构和细节都会在下一档被优化和重构。

---

### 2.3 · `/ppt-narrative-review` — 审论述

**它解决什么问题**：找出你自己看不见的叙事张力问题 + 事实需要核查的点。

**它输出两类**：

1. **叙事张力问题**：比如"第二部分两张相似构图挨在一起，观众会晕，建议拆成两个小节"
2. **事实核查表**：扫出 N 个需要核查的点 —— **它只标位置，不替你下判断**

**关键心智**：review Skill 是放大镜，不是法官。

**完整 review 循环**：

```
review v1
   ↓
P0 三条核查（NotebookLM 内部 + web search · ~10 分钟）
   ↓
结构调整（手动 · Skill 帮不了你）
   ↓
review v2（评分从 7 → 9）
   ↓
[可选] 让 Claude 扮观众，外部视角再审
```

**关键 insight**：现在的 LLM 在「细节工作」上很强，但在「长程思考 + 结构性判断」上是弱点。所以**结构调整是 Skills 替代不了你的那一步**。

## 第 3 章 · 视觉化两条路径

### 路径 A · Claude Design（内测）

- 入口：claude.ai/design → Slide Deck 模式
- 工作方式：上传文字稿 → 它甩 8 个问题逼你做决策 → 生成
- 优势：快、对话式精修、视觉一致
- 限制：内测限流 + 全 web 端 + 不可完全本地控制

### 路径 B · `/visual-deck`（开源 · 推荐对外交付场景）

- 工作流：纯结构版 → 加图片提示词 → Gemini/Flow 出图 → 回喂 Claude 嵌入
- 优势：精细可控 + 版本可追溯 + 不绑定特定出图引擎
- 限制：步骤多，迭代成本高

**怎么选**：
- 内测试用 / 快速看效果 → A
- 对外交付 / 需深度定制 / 需版本控制 → B

## 第 5 章 · 三幕式背后的原理

一份专业方案的灵魂不是视觉，是**三幕式结构**：

1. **研究** —— 一手素材 + 反幻觉防线
2. **分析** —— 骨架、推理、可证伪结论
3. **表达** —— 视觉服务论证

AI 可以每一幕都帮你执行，但它**不能替你串起这三幕**。因为贯穿这三幕的主轴是：

> **你的独立思考。**

研究时：哪条证据可信？分析时：哪条路径最锐？表达时：什么场合全画幅，什么场合分栏？

**这些判断都没法外包。AI 是演员，你是导演。**

这也是为什么 Gamma / Canva 这类一键工具会穿帮 —— 它总是试图把三幕压成一键。研究没做，分析没做，人的独立思考没有位置 —— 漂亮，但是没有底子。

## 第 6 章 · 常见踩坑

### 坑 1 · 在没有证据的情况下让 Skill 写论证

**症状**：Skill 出了一篇看似严谨但全是"幻觉数据"的论证稿。

**修复**：先在 NotebookLM 里**针对每章**拉证据，evidence 文档比你想象的更重要。

### 坑 2 · 把 review Skill 当判官

**症状**：Skill 标出 11 个核查点，你以为它已经判断了，直接 ship。

**修复**：review 只标位置，**真假判断必须你自己来**。P0 三条全流程跑下来 ~10 分钟，跑不掉。

### 坑 3 · 用 Claude Design 触发限流

**症状**：最后一次生成方案的时候卡住、报错、重试。

**修复**：**先把文字稿全部审核完再开 Claude Design**。它不是用来"边想边画"的，是用来"决策已定后视觉落地"的。

### 坑 4 · HTML → PPTX 后换行错位

**症状**：visual-deck 出来的 HTML 在 PPTX 里多出一行换行。

**修复**：在 visual-deck 模板里关掉 `display: block` 的隐式换行，或在转换前手动调整 line-height。
