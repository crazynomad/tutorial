# Mole Cleaner Plus - 方案评审记录

此文件记录了对 `PLAN.md` 初始版本的评审意见和改进建议。

## 📅 评审日期
2026-01-23

## 📊 总体评价
该计划非常合理，不仅可行性高，而且在产品设计和安全性上考虑得非常周全。将“清理”转变为“引导整理”的思路极具人性化。

## ✅ 核心亮点
1.  **产品定位清晰 (Positioning)**: 明确区分了 Mole（系统/开发清理）和 Plus（用户/个人文件整理）的边界。
2.  **高情商设计 (High EQ Design)**: 利用 macOS 原生 **Smart Folder (`.savedSearch`)** 是最优雅的解决方案。它把机器决策变成了用户决策，降低了误删风险。
3.  **痛点精准 (Pain Points)**: 抓住了 iOS 备份和微信缓存这两个普通用户最头疼的磁盘占用大户。
4.  **技术路径可行**: 对 `atime` 不可靠性的识别、保护清单的制定以及 Smart Folder 的技术实现路径都非常清晰。

## 💡 改进建议

### 1. Smart Folder 的生命周期管理
*   **建议**: 生成的 Smart Folder 如果长期堆积在 `~/Library/Saved Searches` 会造成视觉污染。可以考虑将它们统一放在一个父目录下，或在工具中提供“清理生成的搜索”功能。

### 2. 社交软件清理的预期管理
*   **风险**: 清理聊天记录图片后，用户在客户端看到“图片已过期”可能有负面反馈。
*   **建议**: 加强文案提示，并考虑增加一个“仅清理缩略图/临时缓存”的超安全模式。

### 3. 相似照片检测的性能优化
*   **风险**: 对全盘图片计算 pHash 耗时极长。
*   **建议**: 采用分步策略。先利用 Exif 时间信息进行初步筛选（如 1-2 秒内拍摄的照片），仅对这些疑似连拍的照片进行 pHash 计算。

### 4. iOS 备份信息增强
*   **建议**: 在解析 `Info.plist` 时，尽量提取并显示备份对应的 iOS 版本号，帮助用户区分旧设备的备份。

---
**结论**: 方案具备极高的落地价值。建议优先开发 **P0 (Smart Folder)** 和 **P1 (iOS Backups)** 模块。
