# Storyboard 优化建议

> 基于 NotebookLM 笔记本《Claude Code Without the Code》(Nate's Notebook, 64页)

## 一、可直接采用的金句替换

现有脚本已经不错，但 NotebookLM 中有几个表述更锋利，建议替换：

### Scene 06 金句落版 (11:10-11:30)

**现有版本：**
> "Claude Code is not just for coders"
> "它是你的本地 AI 助手"
> "你说人话，它办实事"

**建议替换为（来自 Daniel Nest）：**
> "Claude 是用来**讨论**的，Claude Code 是用来**行动**的"
> "Claude Code is not just for coders"
> "你说人话，它办实事"

**理由**：Talking vs Doing 的对比比 "本地 AI 助手" 更有记忆点，且完美总结了 Scene 06 三组对比的核心（建议 vs 执行）。原来的第一行变成第二行，力度递进更好。

---

### Scene 05 开源生态 (9:50-10:00)

**现有旁白：**
> "这就是 Claude Code 的魔力：让普通人也能用上大神的工具"

**可在旁白后追加一句：**
> "你不需要会写代码，只需要**能清楚地描述你要做什么**"

**来源**：Nate 原文 "Just the ability to describe what you do clearly"。与 Scene 01 TextEmphasis "不需要会写代码" 形成首尾呼应。

## 二、Scene 06 对比框架增强

现有 scene-05-06-plan.md 的三组对比（文件访问 / 操作能力 / 记忆复用）结构已经很好。NotebookLM 提供了更生动的命名和描述，建议融入旁白：

### 6.1 文件访问 → 命名为「失忆症」

**NotebookLM 原文**：Teresa Torres 称云端 AI 为 "Amnesia"（失忆症）——"每次聊天都是从头开始的"

**旁白优化**：
> 现有："网页版在云端，你想让它清理磁盘？它连你的文件都看不到"
>
> 建议：**"网页版有个致命问题——失忆症。每次新开对话，它完全不记得你是谁、你的文件在哪"**

### 6.2 操作能力 → 命名为「动手 vs 动嘴」

**旁白维持不变**（现有的"只能给建议 vs 直接帮你删"已经很好）

可在字幕上加一个标签：
- 左侧加标签 `💬 Talking`
- 右侧加标签 `⚡ Doing`

### 6.3 记忆复用 → 融入「复利效应」

**NotebookLM 原文**：Teresa Torres — "你不再重复自己…你建立的系统会随着时间推移变得更好"

**旁白优化**：
> 现有："Skill 装一次就够了。下次想清理？打开终端说一句话"
>
> 建议追加：**"这不是一次性的对话，而是在构建一个系统。每安装一个 Skill，你的 AI 助手就多学会一项本事"**

## 三、Scene 01 开场可增强的数据弹药

NotebookLM 中有大量 ROI 数据，虽然来自企业场景，但可以佐证"Claude Code 威力"。建议在开场预告中**挑一个**强化冲击力：

**候选数据（任选其一加入 0:35-0:45 过渡段）：**

| 数据点 | 来源 | 适合呈现为 |
|--------|------|-----------|
| 自动化 10,000+ 工单/月，响应提速 65% | Obvi 公司 | 字幕闪过 |
| 6周恢复 $1.2M 销售管道 | 医疗健康平台 | 字幕闪过 |
| 内容系统每周运营成本 15 美分 | 营销自动化案例 | 适合与"省 ¥300"对比 |

**建议用法**：在现有 "今天这期是真正的零基础教程" (0:35) 后快闪 2-3 秒：
> "有人用它自动化了每月一万个客服工单，有人用它恢复了百万美金的销售——而且这些人都不是程序员"

**效果**：拉高期待值，暗示 Claude Code 的能力远不止磁盘清理。

## 五、不建议采用的内容

NotebookLM 中有些内容虽好但**不适合本期视频**：

| 内容 | 不采用理由 |
|------|-----------|
| 并行处理 15 个竞品分析 | 太企业化，与磁盘清理场景脱节 |
