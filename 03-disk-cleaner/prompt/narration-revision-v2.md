# 口播增补修改意见 v2

> 基于作者反馈：开场需要悬念钩子 + Doc-mindmap 结尾需要价值提升

---

## 修改点 1: 开场重录（需要重新录制口播）

### 问题

当前开场是"平铺直叙"式介绍——依次列举三个 Skills，缺乏悬念和留存钩子。观众在前 15 秒内没有足够的理由继续看下去。

### 修改方案

**核心策略**：用"省钱"这个人人都关心的钩子开场，制造好奇心，再用"知识库"这个高价值概念做第二层吸引。

#### 新开场口播（建议重录）

> 哈喽大家好，欢迎回到绿皮火车。
>
> 今天我要用 AI 帮我省下一顿火锅钱。
>
> 而且不是开玩笑——我会用三个免费的 Skills，在十几分钟之内，给我的 Mac 释放 120 个 G 的硬盘空间，还顺便把堆了好几年的 PPT 和 PDF 变成一个随用随查的个人知识库。
>
> 关键是：整个过程零代码，你只需要会打字、会说人话就行。
>
> 不信？咱们直接看效果。

#### 对比说明

| | 旧版 | 新版 |
|---|---|---|
| 钩子 | 无，直接列举 Skills | "省下一顿火锅钱" — 利益驱动 |
| 悬念 | 无 | "不信？" — 制造期待 |
| 价值预告 | 逐个介绍 3 个 Skill | "120G + 个人知识库" — 结果导向 |
| 门槛感 | "0 基础就能上手" | "零代码，会说人话就行" — 更口语化 |
| 节奏 | 90 秒才进入演示 | 30 秒直接开干 |

#### 注意事项

- 旧版开场中关于"第一个/第二个/第三个 Skill"的逐个介绍可以删掉，因为观众在这个阶段并不关心技能名称，他们关心的是"能帮我解决什么问题"
- 旧版中"所有人都能看懂的安装攻略"这段保留价值不大——观众默认教程就是教他们怎么装的，不需要提前说
- "咱们先看看效果。我打开我的 Claude……"这段可以直接衔接新开场，无缝过渡到 Scene 01

## 修改点 2: Doc-mindmap 结尾价值提升

### 问题

当前 Scene 02B.6 在展示完 Finder 分类成果后，结语是：

> "30 份 PPT，从转格式、写摘要、自动分类到重命名，一句话，全部自动完成。困扰我好多年的事，就这么搞定了。"

这段只是在总结"做了什么"，没有提炼出这件事的真正价值。观众看完会觉得"哦，帮我整理了文件"，但不会意识到这其实是一个**个人知识库**的雏形。

### 修改方案

在现有结语后面，追加一段"价值升华"。

#### 新增口播（接在 2B.6 现有内容之后）

> 你想想看，现在我们有了什么？
>
> 按客户查，一秒找到给某个客户做过的所有方案；按主题查，AI 产品管理、数据治理，直接进文件夹；按用途查，要培训材料还是营销方案，分门别类。
>
> 再加上每份文档都有 AI 写的摘要，打开目录就知道里面讲了什么，根本不用一个一个点开。
>
> 这不就是一个最简单、最易检索的个人知识库吗？
>
> 而且它不需要任何特殊的软件，不用装 Notion，不用装 Obsidian，就是普通的文件夹加纯文本。任何电脑都能打开，永远不会过时。
>
> 以前这些 PPT 就是一堆乱码文件名的死文件。现在，它们变成了一个你随时能用、随时能查的活知识库。这才是这个 Skill 真正厉害的地方。

#### 效果对比

| | 修改前 | 修改后 |
|---|---|---|
| 定位 | 文件整理工具 | 个人知识库构建器 |
| 观众感受 | "帮我分了个类" | "帮我把死文件变成了活知识" |
| 价值层次 | 操作层（分类、重命名） | 认知层（检索、复用、知识管理） |
| 记忆点 | 无 | "最简单的个人知识库" |

## 附：对总结部分（Scene 09）的连带调整

如果采纳以上修改，建议同步调整总结部分的第三条：

#### 当前版本

> 第三，今天学会了瘦身、收纳、提炼三步文件管理——磁盘瘦身、文件收纳、文档提炼，一条龙搞定。

#### 建议调整为

> 第三，今天学会了三步文件管理——磁盘瘦身、文件收纳、文档提炼。而且最后那一步，直接帮你把堆了多年的文档变成了一个随用随查的个人知识库。

这样首尾呼应：开场预告了"个人知识库"，结尾再次强化这个概念。
