## Scene 01: 磁盘清理演示

### 1.1 触发 Skill

> 接下来就是重点了。

> 我就像跟朋友抱怨一样，跟它说：我的磁盘空间不够了，帮我清理一下。

> 看，Claude 自动识别到我们需要清理磁盘，然后触发了对应的 Skill。

---

### 1.2 环境检测

> 它现在在干嘛呢？它先检查我们电脑上有没有必要的工具。

> 没有？没关系。它会自己帮我们装，完全不用操心。

---

### 1.3 预览报告

> 好，扫描完了。看这个报告，它把可以清理的东西分门别类列出来了。

> 浏览器缓存 5 个 G，npm、pip 这些开发工具的缓存 8 个 G，AI 模型缓存 3 个 G。这些就是我们平时用各种工具留下的数字垃圾。

> 更贴心的是呢，它还给了三个档位的建议。低风险适合保守派，默认档位推荐大多数人用，最大拯救适合真的特别缺空间的。

---

### 1.4 确认清理

> 注意，它不会擅自行动。它会先问我们：确定要清理吗？这点很重要。

> 我选默认档位，确认。

*(画面：清理进度加速)*

> 清理中……

---

### 1.5 成就展示

> 完成了！看，它自动打开了一个成就页面。

> 释放了 47 个 G。相当于省了 141 块钱。一顿火锅钱到手。

> 还有个彩蛋，它会随机夸一句原作者 tw93。觉得好用的话，可以去 GitHub 给原项目 Mole 点个 Star，支持一下开源大神。

## Scene 02: 文件整理演示

### 2.1 问题场景

> 清理完磁盘，我们再来整理一下文件。

> 打开我们的下载文件夹看看。是不是特别乱？各种 PDF、图片、安装包、截图混在一起。说实话我的下载文件夹简直就是个数字垃圾场。

---

### 2.2 手动模式

> 跟 Claude 说：帮我整理下载文件夹。

> 它会问你要手动模式还是自动模式。我们先试试手动模式。

> 手动模式不会移动任何文件，它会创建 macOS 原生的智能文件夹。

> 看，点开这个 PDF 文件夹，所有 PDF 自动聚合在这里。它其实是一个智能搜索，文件还在原来的位置，不用担心。

---

### 2.3 自动模式

> 如果你想让它真的帮你归类，那就选自动模式。

> 它会先做一个预览，就是 dry run，告诉你它打算怎么分类，但不会真的移动文件。

> 看看分类方案，没问题的话，确认执行。

*(画面：文件自动分类)*

> 所有文件按类型分好了。图片归图片，文档归文档，安装包归安装包。

> 而且它聪明地避开了代码目录和正在使用的文件。安全第一嘛。

---

## Scene 03: 原理讲解

> 刚才我们用一句话就清理了几十个 G 的空间。那背后到底发生了什么？

> 核心引擎是 tw93 开发的 Mole，GitHub 上三万多 Star 的开源项目。它有 clean、analyze、purge 等一系列命令，知道 macOS 上哪些缓存可以安全删除。

*(画面：MoleFeatures 动画)*

> 但 Mole 是个命令行工具，普通人不会用。这就是 Skill 的价值了。

*(画面：SkillArchitecture 动画)*

> 还记得上期说的嘛，Skill 就是一本给 AI 准备的入职手册。我们说人话，Skill 把它翻译成 Mole 的命令。中间这层翻译，就是 Claude Code 在做的事。

*(画面：OpenSourceEco 动画)*

> 这就是 Claude Code 真正的魔力。它把大神的开源工具，通过 Skill 包装起来，让普通人也能一句话调用。你不需要会写代码，只需要能清楚地描述你要做什么。

## Scene 04: Claude.ai vs Claude Code

### 4.1 失忆症

*(画面：分屏对比动画)*

> 有小伙伴可能会问：网页版 Claude 不也挺强的吗？为什么非要用 Claude Code？

> 三个字：够不到。

> 网页版有个致命的问题——失忆症。每次新开对话，它完全不记得你是谁，你的文件在哪。你每次都得把文件拖进去上传。

> 而 Claude Code 运行在我们自己的电脑上，你的文件就是它的上下文。不需要上传，不需要复制粘贴。

---

### 4.2 动嘴 vs 动手

> 第二个区别。网页版只能告诉你——"建议你打开 Finder，找到资源库，手动删除浏览器缓存"。

> Claude Code 呢？直接帮你删了。106 个 G，实打实地释放。

> 一个 Talking，一个 Doing。一个动嘴，一个动手。差距就在这。

---

### 4.3 复利效应

> 更关键的是第三个区别。

> Skill 装一次就够了。下次想清理磁盘？打开终端，说一句话，搞定。

> 网页版呢？你得重新开对话，重新解释你要什么，重新上传文件。每次都从头来。

> 这不是一次性的对话，而是在构建一个系统。每安装一个 Skill，我们的 AI 助手就多学会一项本事。这就是复利。

---

### 4.4 金句

*(画面：全黑背景，文字逐行出现)*

> 所以——

> Claude 是用来讨论的。Claude Code，是用来行动的。

> Claude Code is not just for coders.

> 你说人话，它办实事。

## Scene 05: 安装 Claude Code

> 我们先得安装 Claude Code。打开浏览器，转到 Claude 官网，找到 Claude Code 的安装页面。

> 页面上有一行安装命令，复制它。

> 然后我们打开应用程序，搜索 Terminal，打开这个叫"终端"的 App。

> 别怕这个黑框框啊。它就是另一种和电脑对话的方式，你打字，它执行，就这么简单。

> 把刚才复制的安装命令粘贴进去，回车。

*(画面：安装过程加速)*

> 等它跑完就行。

> 装好以后呢，输入 claude，回车。第一次用会弹出浏览器让你登录，点击授权就好。

> 回到终端，齐活了。

## Scene 06: 安装 Skills

> 现在 Claude Code 装好了，但它还不会清理磁盘。就像上期说的嘛，通用 AI 就是个天才实习生，你得给它喂知识胶囊，它才能变成专家。

> 这个知识胶囊就是 Skill。

> 那我们先告诉 Claude 去哪找这些 Skills。打开浏览器，找到我的 GitHub 仓库，复制链接。

> 回到终端，直接用中文跟它说：帮我安装这个仓库里的 Skills。然后把链接粘上去。

> 它会自己下载、自己配置，我们就等着就行。

*(画面：安装完成)*

> 好，装好了。注意看，我全程没写一行代码。我就是在跟 AI 聊天而已。

## Scene 08: 免费替代方案

*(画面：表格动画)*

> 有同学可能会说：Claude Code 要付费啊，有没有免费的？

> 有。这里给大家推荐两个替代方案。

> 第一个，Open Code。完全开源免费，社区驱动。功能没有 Claude Code 那么全，但基本够用。

> 第二个，Gemini CLI。上期介绍过的 Google 家的产品嘛，有免费额度。想尝鲜的同学可以试试。

> 当然了，如果追求最好的体验和最完善的生态，Claude Code 还是首选。

> 今天教的所有方法，在这三个工具上都能用，因为 Skill 的原理是一样的。

> 如果你还在观望，可以先试试免费的。用了你就知道，这东西是真的香。
