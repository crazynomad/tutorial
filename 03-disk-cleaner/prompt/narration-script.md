## Scene 01: 磁盘清理演示

### 1.1 环境检测

> 它现在在干嘛呢？它先检查我们电脑上有没有必要的工具。

> 没有？没关系。它会自己帮我们装，完全不用操心。

### 1.2 扫描报告

> 好，扫描完了。看这个报告，它把可以清理的东西分门别类列出来了。

![扫描报告：用户应用缓存 79GB、浏览器缓存 37GB、系统日志 22GB](image.png)

> 用户应用缓存将近 80 个 G，浏览器缓存 37 个 G，系统日志 22 个 G。这些就是我们平时用各种软件留下的数字垃圾。

> 下面它给了五个选项。前三个是清理档位：Air 最安全，只清浏览器缓存和日志，能释放 60 个 G；Pro 是推荐档，平衡安全与空间，能释放 140 个 G；Max 是最大化释放，将近 144 个 G。

> 还有两个贴心选项：第四个是先生成一份 CSV 完整清单，让你看清楚到底要删什么；第五个是配置白名单，保护你不想被碰的路径。

### 1.3 刨根问底

> 我先选第四个，看看完整清单。

![CSV 清单已保存到桌面](image-1.png)

> 看，它生成了一份详细的 CSV 列表，直接保存到桌面了。想看具体删了什么，打开这个文件一目了然。

### 1.4 配置白名单

> 接下来我配一下白名单。看，它提供了三种预设模板：办公、开发者、媒体。

![白名单预设模板：office/developer/media](image-2.png)

> 我选 A，办公模式。这样它就会自动保护我的文档和桌面文件夹，清理的时候完全不碰这些地方。

![已添加办公预设，保护 ~/Documents 和 ~/Desktop](image-3.png)

### 1.5 确认清理

> 白名单配好了，现在选清理方案。注意看，它再次确认了我的 Documents 和 Desktop 已受保护，不会被清理。

![三个清理方案 + 白名单保护确认](image-4.png)

> 我选 2，推荐的 Pro 档。

### 1.6 清理完成

> 清理中……

![清理成果：释放 ~106GB，可用空间从 3.3% 提升到 14.8%](image-5.png)

> 完成了！看这个成果：清理前只剩 31 个 G 可用，清理后变成 137 个 G。释放了 106 个 G！磁盘可用空间从 3.3% 直接拉到 14.8%。

> 而且注意看底下——Documents 和 Desktop 都没被碰，白名单生效了。还贴心地提醒你，觉得 Mole 好用的话可以去 GitHub 给原项目点个 Star。

### 1.7 成就页面

> 看，它还自动打开了一个成就页面。

![成就页面：106GB，省了 ¥310.55，一顿火锅钱到手](image-6.png)

> 释放了 106 个 G，相当于省了 310 块钱。现在的 MacBook 升级 1T 硬盘要 3000 块，Skill 帮我释放了这么多空间，相当于一顿火锅钱到手了。

> 还有个彩蛋，它会随机夸一句原作者 tw93——"Mole 小而美，tw93 大而强"。觉得好用的话，可以去 GitHub 给 Mole 点个 Star，支持一下开源大神。

## Scene 02B: 文档智能 + 文件管家演示

> 前面的磁盘清理和文件整理，说实话，如果不怕辛苦，我们自己也能干。手动删缓存、手动建文件夹，无非就是花点时间。

> 但有一件事，一直让我望而却步——就是对这么多年工作中留下来的 PPT 和 PDF 的整理。

> 虽然刚才通过智能文件夹，我们已经有了统一管理的能力。可是电脑上有 4000 多份 PPT，14000 多份 PDF。这里面确实有很多在未来工作中能派上用场的好东西。

> 可说实话，谁有时间一份一份地打开、根据内容去解读和分类呢？这件事困扰我好多年了，一直不敢面对。

> 所以今天要介绍的这个更强大的 Skill——文档智能整理助手，就是来解决这个大难题的。

---

### 2B.1 文档智能：批量转换 + 摘要

> 我跟 Claude 说：帮我把下载文件夹里的 PDF 做个整理分析。

*(录屏：触发 doc-mindmap Skill)*

> 看，它先把 PDF 批量转成了 Markdown 格式，方便后续处理。

> 然后它调用了本地的 Ollama 模型，给每份文档生成了一段摘要。注意，这个过程完全在本地运行，你的文件不会上传到任何地方。

*(录屏：展示转换结果 + 摘要输出)*

### 2B.2 文档智能：分类 + 重命名

> 摘要有了，它接着做分类。从内容主题、文件用途、时间三个维度自动归类。

> 还能智能重命名——把那些 "download(3).pdf" 这种没意义的文件名，根据内容改成有意义的名字。

*(录屏：展示分类结果 + 重命名前后对比)*

### 2B.3 文档智能：思维导图

> 最后，它还能根据文档内容生成思维导图，帮你快速理解一堆文件到底在讲什么。

*(录屏：展示生成的思维导图)*

> 从格式转换到知识提取，一条命令全搞定。

---

### 2B.4 文件管家：一键串联

> 好，前面我们分别演示了清理、整理、文档智能，每次都是单独跟 Claude 说一句话。

> 有没有办法一步到位？有。这就是 file-master。

> 我就说一句：帮我把文件从清理到分析全部搞定。

*(录屏：触发 file-master Skill，展示三阶段自动串联执行)*

> 看，它自动串联了三个步骤：先瘦身，清掉垃圾缓存；再收纳，整理文件分类；最后提炼，做文档智能分析。瘦身、收纳、提炼，一句话启动，三步自动完成。

> 这就是 file-master 的价值——你不用记住三个 Skill 怎么用，一句话就够了。

## Scene 03: 原理讲解

> 刚才我们用一句话就搞定了清理、整理、分析。那背后到底发生了什么？

> 核心引擎是 tw93 开发的 Mole，GitHub 上三万多 Star 的开源项目。它有 clean、analyze、purge 等一系列命令，知道 macOS 上哪些缓存可以安全删除。

*(画面：MoleFeatures 动画)*

> 文档智能这边呢，核心是 doc-mindmap。它能把各种格式的文档转成 Markdown，用本地 Ollama 做摘要，还能自动分类、重命名、生成思维导图。

*(画面：DocMindmapFeatures 动画)*

> 但这些底层工具——Mole、Ollama、macOS 原生命令——普通人都不会用。这就是 Skill 的价值了。

*(画面：SkillArchitecture 动画)*

> 还记得上期说的嘛，Skill 就是一本给 AI 准备的入职手册。我们说人话，Skill 把它翻译成底层工具的命令。中间这层翻译，就是 Claude Code 在做的事。

> 而 file-master 更进一步，它把清理、整理、文档智能三个步骤串成一条流水线。一句话启动，三步自动完成。

*(画面：FileMasterPipeline 动画)*

*(画面：OpenSourceEco 动画)*

> 这就是 Claude Code 真正的魔力。它把大神的开源工具，通过 Skill 包装起来，让普通人也能一句话调用。你不需要会写代码，只需要能清楚地描述你要做什么。

## Scene 04: Claude.ai vs Claude Code

### 4.1 失忆症

*(画面：分屏对比动画)*

> 有小伙伴可能会问：网页版 Claude 不也挺强的吗？为什么非要用 Claude Code？

> 三个字：够不到。

> 网页版有个致命的问题——失忆症。每次新开对话，它完全不记得你是谁，你的文件在哪。你每次都得把文件拖进去上传。

> 而 Claude Code 运行在我们自己的电脑上，你的文件就是它的上下文。不需要上传，不需要复制粘贴。

---

### 4.2 动嘴 vs 动手

> 第二个区别。网页版只能告诉你——"建议你打开 Finder，找到资源库，手动删除浏览器缓存"。

> Claude Code 呢？直接帮你删了。106 个 G，实打实地释放。

> 一个 Talking，一个 Doing。一个动嘴，一个动手。差距就在这。

---

### 4.3 复利效应

> 更关键的是第三个区别。

> Skill 装一次就够了。下次想清理磁盘？打开终端，说一句话，搞定。

> 网页版呢？你得重新开对话，重新解释你要什么，重新上传文件。每次都从头来。

> 这不是一次性的对话，而是在构建一个系统。每安装一个 Skill，我们的 AI 助手就多学会一项本事。这就是复利。
